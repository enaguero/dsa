# Notes — Long-Form Study Material

This directory holds the deep, prose-style references for the repository. Where [`problems/`](../problems/) is "do the exercise," `notes/` is "understand the theory." Everything here is plain Markdown that GitHub renders directly and that Pandoc compiles to PDF via the eisvogel template (see [`docs/BUILDING.md`](../docs/BUILDING.md)).

The top-level [project README](../readme.md) covers both tracks at a glance.

## What's here

| File | What it is |
|---|---|
| `ram_model.md` | The main reference (~100 PDF pages). RAM model + asymptotic analysis + analysis mechanics + algorithm catalog + advanced topics + 60 worked exercises. Begin here. |
| `cheatsheet.md` | Big-O rules, the Master Theorem, complexity tables for data structures and algorithms. Use as a desk reference. |
| `roadmap.md` | Recommended study path through the 150 NeetCode problems. |
| `NEETCODE-150.md` | Canonical problem list with categorization. |
| `figures/` | PNG figures embedded by `ram_model.md`. Generated by `scripts/figures/` and committed so the markdown renders on GitHub without a build step. |
| `slides/` | Beamer slide decks derived from `ram_model.md`, split into parts. |

## Using `ram_model.md`

The document is dense by design but has explicit entry points for different audiences. The document's own §1.1 ("How to use this document") maps these in detail; here is the short version.

### Pick a reading path

- **First-time reader / coming from problem solving.** Start with **§1.2 "A 5-minute quickstart"** — Python `two-sum` example, the three patterns that cover most code you'll write, a "will-it-fit?" table. Then read **§6 → §7 → §11 → §14.9 → §27** in that order; that gives you the working vocabulary in about an hour.
- **Want the theory.** Read Parts I and II in order. Roughly one undergraduate algorithms course's worth of background, compressed.
- **Analyzing your own code.** Skip to Part III. The recipe is in **§13** (worked example first, then the procedure); loop patterns in **§14**; recurrence techniques in **§§16–18**; amortized, space, randomized in **§§19–21**.
- **Looking up a specific algorithm.** Part IV is the catalog: §23 search, §24 sort, §25 graph, §26 DP, §27 recurrence cheat sheet.
- **Beyond the basic model.** Part V (§§28–33): memory hierarchy, external memory, cache-oblivious, parallel models, lower bounds, online/competitive analysis.

### Practice as you go

Each Part ends with a short **"Test your intuition"** sidebar (3–4 quick conceptual checks) whose answers are in **§A**. When you want longer, working-out problems, **§B** holds **60 worked exercises** tagged `[easy] / [med] / [hard]`, distributed across the five Parts:

| Annex section | Topic | Count |
|---|---|---|
| §B.1 | RAM model (Part I) | 6 |
| §B.2 | Asymptotic analysis (Part II) | 12 |
| §B.3 | Mechanics — loops, recurrences, amortized, space, randomized (Part III) | 22 |
| §B.4 | Catalog — sort/search/graph/DP (Part IV) | 12 |
| §B.5 | Beyond RAM — external memory, parallel, lower bounds, competitive (Part V) | 8 |

Solutions are in **§C**, structured so the solution to `B.k.j` is at `C.k.j`. Always attempt the exercise first; consult the solution to verify.

## Building the PDFs

Quick reference (full prerequisites are in `docs/BUILDING.md`):

```bash
# Build ram_model.pdf (the default SRC).
make pdf

# Build any other note.
make pdf SRC=cheatsheet

# Build every note in notes/*.md.
make all

# Build the Beamer slide decks (notes/slides/ram_model_part*.md).
make slides

# Live-rebuild on save (needs fswatch: `brew install fswatch`).
make watch
```

Output PDFs land in `build/`. The `build/` directory is gitignored — only the markdown sources are committed.

## How figures work

The figures embedded in `ram_model.md` are PNG files committed under `notes/figures/`, produced from matplotlib scripts in `scripts/figures/`. Committing the PNGs lets GitHub render the markdown without anyone needing a Python environment; rebuilding is only required when a script changes.

```bash
# Regenerate every notes/figures/*.png from scripts/figures/fig_*.py.
# Needs `uv` (brew install uv) — matplotlib is pulled in transiently.
make figures
```

The build pipeline uses Pandoc's `--resource-path=.:notes` flag (see `Makefile`) so that image references in the markdown can be written relative to the markdown file itself (`figures/foo.png`) and resolve correctly both on GitHub and during PDF compilation.

## Adding a new figure

1. Write `scripts/figures/fig_<your_slug>.py`. Import the shared palette and helpers from `scripts/figures/_style.py` so the figure matches the document's typography and colors. Save with `save(fig, "your-slug")` — the helper writes `notes/figures/your-slug.png` at print-quality DPI.
2. `make figures` to regenerate (this runs `scripts/figures/generate_all.py`, which picks up every `fig_*.py` in the directory).
3. Embed the figure in the markdown with a descriptive caption:
   ```markdown
   ![Caption that names what the figure shows and what to notice.](figures/your-slug.png)
   ```
4. Commit both the script and the PNG.

A few constraints to be aware of when writing figure scripts:

- Matplotlib's mathtext does **not** speak `\le` / `\ge`; use `\leq` / `\geq` instead.
- `set_aspect("equal")` on a wide table-style layout will silently break it; use the default aspect.
- Keep total physical figure width near 6.0–6.6 in so it embeds 1:1 at PDF text width (6.5 in at 1 in margins).

## Adding a new note

1. Drop a new `notes/<topic>.md` file. Match the YAML frontmatter style used by `ram_model.md` (title, subtitle, author, fonts, table of contents).
2. Run `make pdf SRC=<topic>` to verify the PDF compiles cleanly.
3. Once it builds, `make all` will pick it up automatically — `make list` shows every source/PDF pair the Makefile knows about.
4. If your note embeds figures, follow the figure pipeline above.

## How this material was created

The current state of `ram_model.md` reflects an iterative pass with Claude Code in the loop. The work happened in roughly four phases, each captured in commit history:

1. **Theory pass.** Drafted Parts I–V as a graduate-level reference (CLRS-style depth, rigorous definitions, full proofs).
2. **Visual pass.** Replaced every ASCII diagram with a matplotlib-generated figure, infrastructure in `scripts/figures/` (palette and style sharing via `_style.py`, one script per figure). 24 figures total.
3. **Pedagogical pass.** Added scaffolding for first-time readers without disturbing the existing rigor: the §1.2 quickstart, the §13 worked-example-first refactor, the §14.9 pattern dictionary, the §14.10 Python-cost cheat sheet, the §16 reorder (visual method first), and the §A *Test your intuition* sidebars.
4. **Exercise pass.** Added Annexes §B (60 worked exercises) and §C (full solutions), distributed across the five Parts and tagged by difficulty.

Each phase was a separate commit; see `git log --oneline notes/ram_model.md` for the trail.
